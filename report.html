Implementation: Define a Performance Metric¶

It is difficult to measure the quality of a given model without quantifying its performance over training and testing. This is typically done using some type of performance metric, whether it is through calculating some type of error, the goodness of fit, or some other useful measurement. For this project, you will be calculating the coefficient of determination, R2, to quantify your model's performance. The coefficient of determination for a model is a useful statistic in regression analysis, as it often describes how "good" that model is at making predictions.

The values for R2 range from 0 to 1, which captures the percentage of squared correlation between the predicted and actual values of the target variable. A model with an R2 of 0 is no better than a model that always predicts the mean of the target variable, whereas a model with an R2 of 1 perfectly predicts the target variable. Any value between 0 and 1 indicates what percentage of the target variable, using this model, can be explained by the features. A model can be given a negative R2 as well, which indicates that the model is arbitrarily worse than one that always predicts the mean of the target variable.

Question 3 - Training and Testing¶

  • What is the benefit to splitting a dataset into some ratio of training and testing subsets for a learning algorithm?

Hint: Think about how overfitting or underfitting is contingent upon how splits on data is done.

Answer: If we use all the data available for training and then make predictions for any future points, we wont be confident about how our model is performing on those data points. Making a false prediction sometimes may cost big and could jeopardize the project. In order to prevent this, we usually split the data into two sets - training and test sets. Training set is used for training the model and then we use the testing set to evalute how that model is performing by comparing its predictions with the actual labels and finding the error. This allows us to fine-tune our model and evaluate its performance before moving forward with predictions on new data points. Also, if all the available data is used to train the model, then the model tends to memorize all the characteristics of the data and it fails to generalize well to new data which again results in false predictions. This phenomenon is called Overfitting. Another benefit of splitting the data into training and testing set is that we can prevent our model from Overfitting. An optimum ratio of training and testing subsets must be established for the model to perform well and make good predictions on unseen data points.

Answer: Grid search technique is used to determine the optimum values of parameters of a given model. When you have a model which can take various combinations of parameters, grid search technique treats each of this combination a separate model and evaluates the performance of each model. This helps us in determining what values to select for parameters in order to optimize the performance of our model. The combination of parameters used for each model lie on a grid, which is why its called Grid Seach technique. Each model on this grid is tested and evaluated and the best model is selected. If a value of a parameter is integer, then for each combination, this value in increased at a certain scale e.g : 0.1,1,10 ... For example, an SVM classifier can take the parameters - kernel and C. Acceptable values for kernel parameter is 'linear' or 'poly' or 'rbf' and C takes numbers, in this case lets take 0.1, 1, 10. With these parameters, a total of 9 combinations is possible -- kernel = 'linear', C = 0.1; kernel = 'linear', C = 1; kernel = 'linear', C = 10; kernel = 'poly', C = 0.1; .... All these combinations lie on a grid and grid search trains and evaluates all these models. We then select the best model with the highest evaluation metric.

Question 8 - Cross-Validation¶

  • What is the k-fold cross-validation training technique?

  • What benefit does this technique provide for grid search when optimizing a model?

Hint: When explaining the k-fold cross validation technique, be sure to touch upon what 'k' is, how the dataset is split into different parts for training and testing and the number of times it is run based on the 'k' value.

When thinking about how k-fold cross validation helps grid search, think about the main drawbacks of grid search which are hinged upon using a particular subset of data for training or testing and how k-fold cv could help alleviate that. You can refer to the docs for your answer.

Answer: When you have different models with different parameters and deciding which model to use based on the performance results of testing set, it will not give good results for future predictions of unseen points since the parameter values chosen might be good only for that particular testing set and might not work for other future data points. This is overfitting. In order to avoid this, the data is split into three sets - training, validation and testing but this might result in too less data points for training and not enough to make a good model. To avoid this problem of losing data, K-fold cross validation is used. The data is first split into training and testing and the testing set is set aside for final testing. Now the data in the training set is divided into 'k'buckets. We train the model 'k' times each time using a different bucket as testing/cross validation set and the other k-1 buckets as training set. The final performance metric of the model is evaluated by taking the average of all the metrics. We then select the model with highest performance metric and finally test it on the testing set to see if it performs well. This process gives us a more generalised model and prevents us from overfitting which is a possibility when using the Grid Search technique. The Grid Search has a validation set and we evaluate the performance of models with different combinations of parameters using this set. This might result in overfitting since we are using the same set many times. K-Fold cross validation prevents this problem.

from sklearn.tree import DecisionTreeRegressor
from sklearn.metrics import make_scorer
from sklearn.grid_search import GridSearchCV

def fit_model(X, y):
    """ Performs grid search over the 'max_depth' parameter for a 
        decision tree regressor trained on the input data [X, y]. """
    
    # Create cross-validation sets from the training data
    # sklearn version 0.18: ShuffleSplit(n_splits=10, test_size=0.1, train_size=None, random_state=None)
    # sklearn versiin 0.17: ShuffleSplit(n, n_iter=10, test_size=0.1, train_size=None, random_state=None)
    cv_sets = ShuffleSplit(X.shape[0], n_iter = 10, test_size = 0.20, random_state = 0)

    # TODO: Create a decision tree regressor object
    regressor = DecisionTreeRegressor()

    # TODO: Create a dictionary for the parameter 'max_depth' with a range from 1 to 10
    params = {'max_depth': range(1,11)}

    # TODO: Transform 'performance_metric' into a scoring function using 'make_scorer' 
    scoring_fnc = make_scorer(performance_metric)

    # TODO: Create the grid search cv object --> GridSearchCV()
    # Make sure to include the right parameters in the object:
    # (estimator, param_grid, scoring, cv) which have values 'regressor', 'params', 'scoring_fnc', and 'cv_sets' respectively.
    grid = GridSearchCV(regressor,params,scoring_fnc,cv = cv_sets)

    # Fit the grid search object to the data to compute the optimal model
    grid = grid.fit(X, y)

    # Return the optimal model after fitting the data
    return grid.best_estimator_
